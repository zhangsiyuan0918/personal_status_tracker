# 《个人状态记录器 v0.1.5 PRD》

版本：v0.1.5  
产品名：个人状态记录器  
版本定位：**记录闭环版 / 最近记录概览版**  
基于版本：v0.1.4  
目标平台：移动端优先 PWA / Web App  
核心目标：在 Dashboard 中提供最近 7 天记录概览，让用户快速知道哪些天已记录、哪些天未记录，并能点击某一天直接补记或编辑，形成记录闭环。

---

# 1. 版本背景

v0.1.4 已经实现：

- 最近 30 天任意日期补记；
- 今天/昨天快捷补记；
- 日期选择器；
- 历史记录编辑对应日期；
- 保存到指定日期。

这解决了一个重要问题：

> 用户可以补记更早的日期。

但新的问题是：

> 用户需要知道哪些日期需要补记。

如果用户不知道最近哪几天没记，就需要手动回忆或进入历史记录查看，这仍然有阻力。

因此 v0.1.5 的重点是：

> 在 Dashboard 上展示最近 7 天的记录概览，并让每一天都可以点击进入记录页补记或编辑。

---

# 2. 当前问题

## 2.1 补记入口不够直接

虽然 v0.1.4 支持选择任意日期补记，但用户需要：

1. 进入记录页；
2. 打开日期选择器；
3. 想起自己漏了哪一天；
4. 选择日期；
5. 再记录。

这个流程仍有认知成本。

---

## 2.2 用户不知道哪些天没记

当前只有历史记录和趋势图，但没有一个轻量入口告诉用户：

```text
最近 7 天哪些天已记录，哪些天未记录。
```

---

## 2.3 Dashboard 缺少记录闭环

Dashboard 目前主要展示今日状态和覆盖率，但还不能直接完成：

```text
看到漏记 → 点击日期 → 补记
```

这个闭环。

---

# 3. v0.1.5 核心目标

v0.1.5 的目标：

1. 在 Dashboard 展示最近 7 天记录概览；
2. 每一天显示已记录 / 未记录；
3. 已记录日期显示主状态；
4. 未记录日期显示「未记录」；
5. 点击任意日期可进入 RecordPage；
6. 已记录日期进入编辑；
7. 未记录日期进入补记；
8. 保留最近 7 天覆盖率；
9. 不做完整日历、不做周复盘、不做复杂统计。

---

# 4. 设计原则

## 4.1 让补记变得可见

如果用户可以补记，就应该能看到哪些天可以补。

---

## 4.2 不制造打卡焦虑

不使用：

- 断签；
- 连续打卡；
- 漏打；
- 失败；
- streak。

使用更温和的说法：

```text
最近 7 天已记录：X / 7 天
```

---

## 4.3 点击即进入对应日期

用户不应再手动选择日期。

看到某天未记录，直接点击即可补记。

---

## 4.4 轻量概览，不做完整日历

本版本只展示最近 7 天。  
完整月历留给 v0.2 或之后。

# 7. 用户场景

## 场景 1：发现昨天没记

用户打开 Dashboard，看见最近 7 天概览中昨天显示「未记录」。

操作：

1. 点击昨天卡片；
2. 进入 RecordPage；
3. 日期自动设置为昨天；
4. 用户补记；
5. 保存后返回 Dashboard 或留在 RecordPage。

---

## 场景 2：编辑前几天记录

用户发现 3 天前已经记录，但想修改。

操作：

1. 点击最近 7 天概览中的那一天；
2. 进入 RecordPage；
3. 自动加载该日期记录；
4. 用户修改；
5. 保存覆盖。

---

## 场景 3：今天还没记录

Dashboard 显示：

```text
今天还没有记录。
```

同时最近 7 天中的今天卡片显示：

```text
今天 未记录
```

用户点击今天卡片或主按钮即可记录。

---

## 场景 4：快速查看最近状态分布

用户打开 Dashboard，快速看到：

```text
最近几天状态大多是平稳运行，但有两天连接少了一点。
```

不需要进入趋势图。

# 8. 核心功能详述

---

# 8.1 最近 7 天记录概览

## 8.1.1 功能目标

在 Dashboard 展示最近 7 天的记录状态。

---

## 8.1.2 展示位置

建议放在 Dashboard 今日状态模块下方。

页面结构：

```text
今日状态
...

最近 7 天
[日期卡片列表]

最近 7 天已记录：X / 7 天
```

---

## 8.1.3 日期范围

最近 7 天包括今天。

例如今天是：

```text
2026-05-19
```

则展示：

```text
2026-05-13
2026-05-14
2026-05-15
2026-05-16
2026-05-17
2026-05-18
2026-05-19
```

---

## 8.1.4 排序方式

推荐按时间升序：

```text
较早日期 → 今天
```

原因：

- 更像一条时间线；
- 用户容易看到最近几天状态变化。

---

# 8.2 每日卡片内容

## 8.2.1 已记录日期

如果该日期已有记录，卡片显示：

```text
05/15
进入状态了
```

或：

```text
05/15
已记录
进入状态了
```

建议简洁显示：

```text
05/15
进入状态了
```

---

## 8.2.2 未记录日期

如果该日期无记录，卡片显示：

```text
05/15
未记录
```

---

## 8.2.3 今天标识

如果日期是今天，卡片增加标识：

```text
今天
```

例如：

```text
今天
05/19
未记录
```

或：

```text
05/19
今天 · 未记录
```

---

## 8.2.4 主状态

已记录日期使用：

```ts
getPrimaryStatus(record)
```

返回主状态名称。

例如：

```text
平稳运行
有点没电
进入状态了
连接少了一点
脑子有点吵
没什么想做
```

---

# 8.3 状态颜色

## 8.3.1 功能目标

让用户快速通过颜色识别状态。

---

## 8.3.2 建议颜色

| 状态 key | 状态名称 | 颜色 |
|---|---|---|
| recovery | 有点没电 | 灰蓝 |
| rumination | 脑子有点吵 | 玫红/红 |
| low_burn | 没什么想做 | 橙色 |
| isolation | 连接少了一点 | 紫色 |
| ignited | 进入状态了 | 绿色 |
| stable | 平稳运行 | 蓝色 |
| 无记录 | 未记录 | 深灰 |

---

## 8.3.3 实现方式

可以在状态规则中增加颜色字段，或单独维护映射：

```ts
export const STATUS_COLORS = {
  recovery: "#64748B",
  rumination: "#F43F5E",
  low_burn: "#F59E0B",
  isolation: "#A78BFA",
  ignited: "#10B981",
  stable: "#60A5FA",
  empty: "#374151"
};
```

# 8.4 点击日期卡片

## 8.4.1 功能目标

点击任意日期卡片后，进入 RecordPage 并选中该日期。

---

## 8.4.2 点击已记录日期

流程：

```text
点击已记录日期
↓
切换到 RecordPage
↓
selectedDate = 该日期
↓
加载该日期记录
↓
用户编辑
```

---

## 8.4.3 点击未记录日期

流程：

```text
点击未记录日期
↓
切换到 RecordPage
↓
selectedDate = 该日期
↓
无记录，使用默认值 5
↓
用户补记
```

---

## 8.4.4 技术实现建议

如果当前项目是 Tab state：

在 App 或 DashboardPage 中提供回调：

```ts
onSelectRecordDate(date: string)
```

点击日期时：

```ts
onSelectRecordDate(date)
```

父组件：

```ts
setRecordPageDate(date)
setActiveTab("record")
```

RecordPage 接收：

```ts
initialDate?: string
```

或共享状态：

```ts
selectedRecordDate
```

- 点击已记录日期能进入编辑；
- 点击未记录日期能进入补记；

- 数据加载正确；

# 17. 错误与边界处理

## 17.1 无任何记录

Dashboard 中最近 7 天概览仍展示 7 个日期，全部为未记录。

覆盖率：

```text
最近 7 天已记录：0 / 7 天
```

文案：

```text
先不用追求连续，留下几个状态点就有价值。
```

---

## 17.2 只有一条记录

对应日期显示状态，其余显示未记录。

---

## 17.3 记录日期超出最近 7 天

不显示在最近 7 天概览中，但历史记录和趋势图仍正常。

---

## 17.4 getPrimaryStatus 返回异常

如果状态计算异常，兜底显示：

```text
已记录
```

但正常情况下不应发生。
