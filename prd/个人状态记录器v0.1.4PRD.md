# 《个人状态记录器 v0.1.4 PRD》

版本：v0.1.4  
产品名：个人状态记录器  
版本定位：**日期补记版**  
基于版本：v0.1.3  
目标平台：移动端优先 PWA / Web App  
核心目标：支持用户补记最近一段时间内的任意日期，而不再只限于今天和昨天。

---

# 1. 版本背景

v0.1.3 已经完成了快速记录体验优化：

- 状态片段分组选择；
- 详细分数默认折叠；
- 补充文字默认折叠；
- 状态名称更自然；
- 记录流程更轻量。

但实际使用中仍然存在一个问题：

> 用户不一定每天都会记。  
> 目前只能补记今天和昨天，如果漏记更早的日期，就无法方便补上。

对于个人状态记录工具来说，漏记是很正常的事情。  
产品不应该假设用户每天都能稳定打开，而应该允许用户在想起来的时候补上大概状态。

因此 v0.1.4 的重点是：

> **把「今天/昨天」补记能力升级为「最近 30 天任意日期补记」。**

---

# 2. 当前问题

## 2.1 补记范围太窄

当前只支持：

```text
今天
昨天
```

如果用户漏记了前天、三天前、上周某一天，就无法直接补记。

---

## 2.2 忘记记录后容易中断

如果用户连续几天没记，打开应用后可能会觉得：

> “反正已经断了。”

虽然产品不强调连续打卡，但无法补记会让数据断层更明显。

---

## 2.3 某些有代表性的状态无法补录

有些天可能特别有意义，比如：

- 某天状态特别好；
- 某天很没电；
- 某天进入状态；
- 某天和人产生了连接；
- 某天明显脑子很乱。

如果当时没记录，之后想补上，目前不方便。

---

# 3. v0.1.4 核心目标

v0.1.4 的目标：

1. 支持选择最近 30 天内任意日期进行补记；
2. 保留「今天」「昨天」快捷入口；
3. 选择某个日期后，自动加载该日期已有记录；
4. 如果该日期没有记录，则使用默认值；
5. 保存时写入对应日期；
6. 历史记录支持编辑并跳转到对应日期；
7. 不改变现有数据结构；
8. 不增加复杂日历、批量补记等功能。

---

# 4. 设计原则

## 4.1 允许漏记

用户漏记不是失败。  
产品需要允许用户之后补上。

---

## 4.2 补记不要求精确

补记本身依赖回忆，因此文案要降低压力：

```text
补记不用太精确，留下大概状态就够了。
```

---

## 4.3 范围有限，避免乱选

v0.1.4 只允许补记最近 30 天。

原因：

- 太久远的状态记忆不可靠；
- 限制范围可以减少误选；
- 最近 30 天已经覆盖大部分真实补记场景；
- 实现简单。

---

## 4.4 不做复杂日历

本版本只做日期选择器，不做日历视图。

日历视图留给 v0.2 或之后版本。

---

# 7. 用户场景

## 场景 1：补记前天

用户今天打开应用，发现前天忘记记录。

操作：

1. 进入记录页；
2. 点击「选择日期」；
3. 选择前天日期；
4. 系统提示该日期还没有记录；
5. 用户点选状态片段；
6. 保存。

---

## 场景 2：编辑上周某一天

用户查看历史记录时，发现某天记录不准确。

操作：

1. 进入历史记录；
2. 点击某条记录的「编辑」；
3. 系统进入记录页；
4. 自动选中该记录日期；
5. 加载原记录；
6. 用户修改后保存。

---

## 场景 3：今天正常记录

用户当天记录。

操作：

1. 打开记录页；
2. 默认日期为今天；
3. 点状态片段；
4. 保存。

---

## 场景 4：昨天快捷补记

用户昨天忘记记录。

操作：

1. 进入记录页；
2. 点击「昨天」快捷按钮；
3. 系统选中昨天日期；
4. 用户补记并保存。

---

# 8. 核心功能详述

---

# 8.1 记录日期选择升级

## 8.1.1 功能目标

将原来的：

```text
记录日期：
[今天] [昨天]
```

升级为：

```text
记录日期：
[今天] [昨天] [选择日期]

当前记录日期：2026-05-15
```

---

## 8.1.2 日期选择方式

使用原生日期选择器：

```html
<input type="date" />
```

原因：

- 实现简单；
- 移动端原生体验较好；
- 不引入额外组件库；
- 适合 v0.1.4 轻量迭代。

---

## 8.1.3 默认日期

进入 RecordPage 时默认选择：

```text
今天
```

即：

```ts
selectedDate = getTodayString()
```

---

## 8.1.4 快捷按钮

保留两个快捷按钮：

```text
今天
昨天
```

点击「今天」：

```ts
selectedDate = getTodayString()
```

点击「昨天」：

```ts
selectedDate = getYesterdayString()
```

---

## 8.1.5 选择日期按钮

可以有两种实现方式。

### 方案 A：直接展示日期选择器

```text
选择日期：
[2026-05-15]
```

### 方案 B：点击按钮后显示日期选择器

```text
[选择日期]
```

点击后展开：

```text
<input type="date" />
```

v0.1.4 推荐方案 A，更直接、实现简单。

---

# 8.2 日期范围限制

## 8.2.1 功能目标

限制用户只能选择最近 30 天内的日期，不能选择未来日期。

---

## 8.2.2 日期范围

允许范围：

```text
今天往前 30 天 至 今天
```

例如今天是：

```text
2026-05-15
```

则允许选择：

```text
2026-04-15 至 2026-05-15
```

---

## 8.2.3 input 设置

```tsx
<input
  type="date"
  min={getDateDaysAgoString(30)}
  max={getTodayString()}
/>
```

---

## 8.2.4 越界处理

虽然 input 设置了 min/max，但仍需要在逻辑层校验。

如果用户选择未来日期或超过 30 天前日期，提示：

```text
只能补记最近 30 天内的状态。
```

并且不更新 `selectedDate`，或将其重置到合法日期。

---

## 8.2.5 验收标准

- 不能选择未来日期；
- 不能选择超过 30 天前日期；
- 非法日期不会保存；
- 非法选择有提示。

---

# 8.3 日期切换后的记录加载

## 8.3.1 功能目标

用户切换日期时，系统自动加载该日期的记录。

---

## 8.3.2 有记录时

如果该日期已有记录：

1. 填充表单数值；
2. 填充补充记录；
3. 如果有状态片段字段，可尝试恢复选中态；
4. 显示提示：

```text
正在编辑这一天的记录。
```

---

## 8.3.3 无记录时

如果该日期没有记录：

1. 所有指标默认值为 5；
2. 状态片段无选中；
3. 补充文字为空；
4. 显示提示：

```text
这一天还没有记录。可以凭印象补一个大概状态。
```

---

## 8.3.4 日期较早时的低压力文案

如果选择的日期不是今天，并且不是昨天，可以显示：

```text
补记不用太精确，留下大概状态就够了。
```

---

## 8.3.5 验收标准

- 切换到已有记录日期时，能加载旧记录；
- 切换到无记录日期时，能重置为默认值；
- 补充记录和分数同步变化；
- 不会把当前未保存内容错误写到其他日期；
- 切换日期后保存写入正确日期。

---

# 8.4 保存逻辑

## 8.4.1 功能目标

保存时根据当前选择日期写入记录。

---

## 8.4.2 保存字段

保存时：

```ts
record.date = selectedDate
record.id = selectedDate
```

---

## 8.4.3 新增记录

如果该日期无记录：

```ts
createdAt = now
updatedAt = now
```

---

## 8.4.4 更新记录

如果该日期已有记录：

```ts
createdAt = existingRecord.createdAt
updatedAt = now
```

---

## 8.4.5 保存成功提示

根据日期显示不同提示。

### 今天

```text
已保存今日状态
```

### 昨天

```text
已保存昨天状态
```

### 更早日期

```text
已保存 2026-05-15 的状态
```

---

## 8.4.6 保存后反馈

保留 v0.1.3 的保存后即时反馈：

- 当前状态名称；
- 状态提示文案；
- 查看面板按钮。

如果保存的是非今日记录，反馈卡片中仍显示该日期状态。

例如：

```text
已保存 2026-05-10 的状态

当前状态：进入状态了

今天有明显投入感。可以记一下是什么让你进入了这个状态。
```

注：这里的提示文案中如果有“今天”，可接受；若想更严谨，可统一改成“这一天”。

v0.1.4 推荐轻量处理，不强制修改所有提示。

---

# 8.5 历史记录编辑

## 8.5.1 功能目标

用户可以从历史记录直接编辑某一天。

---

## 8.5.2 当前情况

如果 v0.1/v0.1.3 已有编辑功能，需要增强为：

> 点击编辑后进入 RecordPage，并选中该记录日期。

如果没有编辑功能，则新增「编辑」按钮。

---

## 8.5.3 交互规则

历史记录每条记录增加或保留：

```text
编辑
```

点击后：

1. 切换到 RecordPage；
2. 设置 `selectedDate = record.date`；
3. 加载该日期记录；
4. 用户可修改并保存。

---

## 8.5.4 技术实现建议

如果当前项目使用 Tab state 管理页面，可以在 App 层增加：

```ts
editingDate?: string
```

或：

```ts
initialRecordDate?: string
```

当用户点击历史编辑时：

```ts
setSelectedTab("record")
setRecordPageDate(record.date)
```

如果使用 React Router，可以跳转：

```text
/record?date=2026-05-15
```

v0.1.4 不强制具体实现，只要求功能可用。

---

## 8.5.5 验收标准

- 历史记录有编辑入口；
- 点击编辑后进入记录页；
- 记录页选中对应日期；
- 表单加载对应记录；
- 修改保存后历史记录更新；
- 不影响其他日期记录。

# 10. 数据结构变化

v0.1.4 不改变 `StateRecord` 数据结构。

继续使用：

```ts
export interface StateRecord {
  id: string;
  date: string;

  energy: number;
  spark: number;
  action: number;
  connection: number;
  expression: number;
  stability: number;

  positiveNote?: string;
  drainNote?: string;
  actionNote?: string;

  mode?: "full" | "simple";
  templateKey?: string;
  quickStateKeys?: string[];

  createdAt: string;
  updatedAt: string;
}
```

说明：

- `date` 仍为唯一日期；
- `id` 仍等于 `date`；
- 不新增补记标识；
- 不区分补记和当天记录。

# 12. 状态片段与日期切换的关系

## 12.1 切换到已有记录

如果已有记录有 `quickStateKeys`：

- 可以恢复对应选中态。

如果没有 `quickStateKeys`：

- 不强制反推状态片段；
- Slider / 当前数值正确即可；
- 状态片段可以保持未选中。

v0.1.4 推荐：

> 不做复杂反推，只保证数值加载正确。

---

## 12.2 切换到无记录日期

默认：

```ts
energy = 5
spark = 5
action = 5
connection = 5
expression = 5
stability = 5
```

状态片段选中态清空。

---

## 12.3 切换日期前有未保存内容

v0.1.4 可以简单处理：

> 切换日期不做未保存提醒。

原因：

- 实现简单；
- 当前个人使用风险低；
- 避免交互复杂。

P2 可考虑增加：

```text
当前修改尚未保存，确认切换日期？
```

# 15. 验收标准

## 15.1 日期选择

- 记录页默认选择今天；
- 可以点击今天快捷按钮；
- 可以点击昨天快捷按钮；
- 可以选择最近 30 天内任意日期；
- 不能选择未来日期；
- 不能选择超过 30 天前日期。

---

## 15.2 记录加载

- 选择已有记录日期时，表单正确加载；
- 选择无记录日期时，表单重置为默认值；
- 补充文字正确加载；
- 详细分数正确显示；
- 状态片段不会错误影响加载数据。

---

- 保存今天显示「已保存今日状态」；
- 保存昨天显示「已保存昨天状态」；

- 保存到正确日期；
- 同日期重复保存会覆盖；
- 不同日期记录互不影响；
- createdAt/updatedAt 逻辑正确。
